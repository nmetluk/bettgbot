# TASK-088: Привести реализацию админки в соответствие дизайн-макету (PIN Voice)

## Контекст

Дизайн-аудит сравнил боевую реализацию (`src/admin/static/css/*` + Jinja-шаблоны
+ Bootstrap 5.3.3; `volt.css` — пустой placeholder) с исходным дизайн-макетом, на котором
она строилась (`tokens.css`/`app.css` помечены «Based on …/admin/styles.css»). Реализация
расходится с макетом на трёх уровнях:

1. **Глобальные стили** — часть отклонений осознанная (a11y, TASK-072: тёмный primary
   `#287EA8`, бордеры `#8E8E96`), часть — недоведённые примитивы (бейджи, аватары, переключатель).
2. **Меню и оболочка** — пропали счётчики в сайдбаре, перестроен топбар, добавлены страницы без макета.
3. **Главное — целые семейства компонентов ссылаются на CSS-классы, которых нет в `app.css`/`tokens.css`**
   (Bootstrap их не покрывает) → в проде рендерятся как голый HTML: пагинация, алерты,
   прогресс-бары, хлебные крошки, KPI на Аналитике, половина вариантов бейджей, фильтр-чипы.

Это и есть основная причина ощущения «не похоже на макет». Полный разбор (визуальное
сравнение + таблицы находок A/M/C + постраничный) — в дизайн-проекте, артефакт
`audit/Design-Audit.html`. Ключевые находки воспроизведены ниже, задача самодостаточна.

## Аудит-приёмка (живой прогон в браузере, 2026-05-31)

Cowork-агент прокликал прод `a.pinbetting.ru` по всем разделам и **подтвердил вживую** часть
находок этой задачи: C3 (Аналитика — KPI голым текстом без карточек, заголовок графика без
стиля — самый заметный дефект), статус «ЗАКРЫТ» красным вместо янтаря, активная вкладка
события красная вместо `тёмный+accent`, аватары единым бледно-розовым (A2), тоггл синий
44×24 вместо зелёного 36×20 (A5), крошки plain вместо `pv-breadcrumb` (C7), пустые состояния
без иконки на users/leaderboard/broadcasts (C9), переключатель периода рейтинга без
группы-контейнера (C10).

**Не удалось проверить вживую** (остаются по коду в рамках этой задачи): пагинация C1 (мало
данных), а также алерт/прогресс/бейджи рассылок C2/C4/C5 — список рассылок пуст, **а форма
`/broadcasts/new` падает в HTTP 500** (вынесено в [TASK-089](TASK-089-broadcasts-new-500.md)).

**Важно — разделение со «сломанным функционалом».** Этот таск чисто про CSS/разметку. Во
время прогона нашлись **функциональные баги, вынесенные отдельно** и НЕ входящие в скоуп 088:

- `/broadcasts/new` → HTTP 500 — [TASK-089](TASK-089-broadcasts-new-500.md).
- `Alpine.store('ui')` не регистрируется → не работают тёмная тема, плотность, акцент и
  сворачивание меню (кнопка тоггла ещё и без иконки) — [TASK-090](TASK-090-alpine-ui-store-init.md).
- Аудит: раскрытие деталей записи не работает (`hx-target` на несуществующий id) + глобальный
  поиск в топбаре декоративный — [TASK-091](TASK-091-audit-details-and-search.md).

Поэтому 088 помечен `blockedBy: [TASK-090, TASK-091]`: TASK-090 и TASK-091 правят те же
`_layout_shell.html`/топбар, и косметику тоггла (A5) бессмысленно делать раньше, чем тоггл
вообще оживёт. Берём 088 последним из этой пачки.

## Цель

Устранить системные расхождения с макетом: вернуть/определить недостающие CSS-классы,
унифицировать именования (бейджи, пустые состояния, крошки), починить разметку, завязанную
на несуществующие классы/атрибуты, и синхронизировать оболочку (сайдбар/топбар) с макетом.

### Блок C — классы без стилей (High; чинить в первую очередь)

- [ ] **C1 — Пагинация.** Определить в `app.css` `.pv-pagination`, `.pv-pagination-current`,
      `.pv-pagination-item`, `.pv-pagination-active` (кнопочный ряд, центрированный, токены `--pv-r-6`,
      `--pv-border`). Проверить рендер на: `events/list.html`, `users/list.html`, `audit/list.html`,
      `broadcasts/list.html`. (В events используется ещё `pv-toolbar`-обёртка пагинации — оставить
      один способ.)
- [ ] **C2 — Варианты бейджей.** Сейчас в `app.css` есть только `success/danger/neutral/draft/open/closed`.
      Шаблоны используют ещё `info`, `warning`, `secondary`, `muted` (Bootstrap-нейминг) → серый бейдж.
      Свести к **одному** набору вариантов и определить все используемые: добавить `.pv-badge.info`,
      `.pv-badge.warn`/`.warning`, `.pv-badge.secondary`(=neutral), `.pv-badge.muted`(=neutral).
      Затронутые шаблоны: `broadcasts/list.html` (secondary/info/warning), `analytics/list.html`
      (warning/secondary), `leaderboard/list.html` (warning/secondary), `users/detail.html` (info/muted).
- [ ] **C3 — Аналитика на чужих классах.** `analytics/list.html` использует несуществующие
      `pv-kpi-row`, `pv-kpi-card`, `pv-card-header`, `pv-card-title`. Привести разметку к остальным
      страницам: KPI → `.pv-kpi` внутри `.pv-grid c4`; заголовки карточек → `.pv-card-head` + `<h3>`.
      (✅ подтверждено вживую — KPI рендерятся голым текстом в столбик, это самый заметный дефект.)
- [ ] **C4 — Алерт.** `broadcasts/list.html` использует `pv-alert`/`pv-alert-success`. Либо
      определить компонент алерта в `app.css`, либо переиспользовать карточку-уведомление (как
      на `categories/list.html`). Выбрать один подход на всю админку.
- [ ] **C5 — Прогресс-бар.** `broadcasts/list.html` использует `pv-progress`/`pv-progress-bar` —
      определить (пилюля `border-radius: 99px`, заполнение цветом статуса).
- [ ] **C6 — Фильтр-чип.** `audit/list.html` использует `.pv-chip` («Период») — перенести стиль
      из макета (`1px dashed var(--pv-border-2)`, при `.active` — сплошной).
- [ ] **C7 — Хлебные крошки.** `users/detail.html` использует `pv-crumbs`/`pv-crumbs-sep`/
      `pv-crumbs-current`; в макете класс называется `.pv-breadcrumb`. На `events/form.html` крошек
      нет вовсе («← События»). Унифицировать: один класс крошек, применить на обеих карточках.
      (✅ подтверждено вживую на `users/detail` — крошки plain-ссылкой.)

### Блок C — мелочь (Medium/Low)

- [ ] **C8 — Атрибуты на бейджах.** В `users/list.html` и `users/detail.html` точка/иконка заданы
      атрибутами `data-dot`/`data-icon`, а CSS завязан на **класс** `.dot` и вложенную иконку →
      не отображаются. Заменить `data-dot` → `class="… dot"`; «сбылся/не сбылся» — вложить
      `<span class="material-symbols-rounded">check/close</span>` внутрь бейджа.
- [ ] **C9 — Пустые состояния.** Иконку через `data-icon` рисует только `.pv-empty-state`
      (используется лишь на Аналитике). Остальные (`users`, `audit`, `leaderboard`, `broadcasts`,
      `users/detail`) используют `.pv-empty` + `data-icon` → иконки нет. Свести к одному классу
      пустого состояния с иконкой. (✅ подтверждено вживую на users/leaderboard/broadcasts.)
- [ ] **C10 — `pv-toolbar-toggle`** (`leaderboard/list.html`, переключатель периода) — определить
      контейнер-группу или заменить на существующий паттерн. Косметика. (✅ подтверждено вживую —
      кнопки периода без группы-контейнера; сам переключатель функционально работает.)

### Блок A — глобальные примитивы

- [ ] **A2 — Аватары.** Перенести из макета 8-цветную палитру `.pv-avatar.a0–a7` (заливка цветом +
      белые инициалы), статусы `.online/.busy/.away/.offline` и размеры `sm/lg`. Сейчас один стиль
      (бледно-розовый `--pv-accent-soft`). Хеш-функция выбора цвета по seed — на стороне шаблона/хелпера.
      (✅ подтверждено вживую — все аватары одним бледно-розовым.)
- [ ] **A3 — Бейджи.** Вернуть форму-пилюлю (`border-radius: 99px`) и убрать `text-transform: uppercase`
      (макет — обычный регистр). Затрагивает все страницы.
- [ ] **A5 — Переключатель.** Привести `.pv-toggle` к макету: 36×20px, во включённом — зелёный
      (`--pv-green`), а не синий 44×24. (Либо осознанно зафиксировать новый размер в `docs/adr/0005`.)
      ⚠️ **Зависит от [TASK-090](TASK-090-alpine-ui-store-init.md):** сейчас тоггл темы функционально
      мёртв (стор `ui` не зарегистрирован). Стиль править ТОЛЬКО после оживления стора, иначе
      проверить включённое (зелёное) состояние нельзя. (✅ синий 44×24 подтверждён вживую.)
- [ ] **A6 — `pv-card-flush`.** В макете это `padding:0; overflow:hidden` (рамка сохраняется,
      убираются отступы для таблиц во всю ширину). В реализации — `background:transparent; border:none`
      (карточка теряет рамку). Выровнять с макетом.
- [ ] **A7 — Плотность таблиц.** Ячейки `.pv-table td` фикс. `10px 12px` игнорируют `--pv-row-h`/
      `--pv-cell-y`. Вернуть density-токены, чтобы переключатель C/R/K влиял на таблицы.
      ⚠️ Переключатель плотности тоже завязан на стор `ui` (см. [TASK-090](TASK-090-alpine-ui-store-init.md)) —
      проверять density-токены после его фикса.
- [ ] **A4 (sync) — Focus-ring.** `--pv-focus` всё ещё `rgba(54,169,225,.28)` (старый синий) —
      привести к новому primary `#287EA8`. Сам тёмный primary и тёмные бордеры (TASK-072) **оставить**.
