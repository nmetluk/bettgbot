# Решения — task-004-review

| # | Решение | Альтернативы | Обоснование | Где зафиксировано |
|---|---|---|---|---|
| 1 | `extra="ignore"` в `Settings`-моделях | (а) `extra="forbid"` + добавить `POSTGRES_USER`/`POSTGRES_PASSWORD`/`POSTGRES_DB` как pass-through поля; (б) разделить `.env` на app-`.env` и compose-`.env` | Защита от опечаток через `forbid` полезна, когда `.env` целиком про приложение. У нас `.env` — это микс (app + compose); сосуществование принципиально, и `ignore` корректно отражает реальность. Pass-through поля — холостая работа, дублирующая `DATABASE_URL`. Разделение файлов усложнит UX без выигрыша | [`state/DECISIONS.md`](../../state/DECISIONS.md), [`src/shared/config.py`](../../src/shared/config.py) |
| 2 | Тесты конфига — `monkeypatch.setenv`, не `_env_file=tmp` | (а) Tmp-`.env` файл (как было в DoD); (б) очищать env через `monkeypatch.delenv` перед каждым `_env_file=tmp` тестом | Спецификация задачи требовала tmp-файл, но `conftest.py` со stub-env (нужный для module-level импорта `settings`) перекрывает `_env_file`. `monkeypatch.setenv` — рекомендованный pydantic-settings способ, явнее и контролируемее. Tmp-`.env` нужен был бы только при тестировании самого file-парсинга — не наш use case | [`state/DECISIONS.md`](../../state/DECISIONS.md), [`tests/unit/test_config.py`](../../tests/unit/test_config.py) |
| 3 | Приоритет источников: `os.environ` > `.env`-файл (дефолт pydantic-settings) | Кастомизация через `customise_sources` для явной фиксации | Дефолтный приоритет — де-факто стандарт всей экосистемы (docker env_file → environment override, k8s env, systemd EnvironmentFile). Явная кастомизация без причины — карго-культ; добавит код, который придётся поддерживать | [`state/DECISIONS.md`](../../state/DECISIONS.md), [`src/shared/config.py`](../../src/shared/config.py) |
| 4 | Демо-точка `python -m src.shared.config` остаётся с runpy-warning | Подавлять warning через `-W ignore::RuntimeWarning`; вынести демо в `scripts/dump_config.py`; убрать | Warning не влияет на вывод, не виден в типичной интерактивной работе (`uv run` его не печатает). Подавление через флаг — обходной путь, который маскирует и реальные warning'и. Вынесение в `scripts/` — потеря удобства `python -m`. Принимаем как есть | [`handoff/outbox/TASK-004-report.md`](../../handoff/outbox/TASK-004-report.md) |
