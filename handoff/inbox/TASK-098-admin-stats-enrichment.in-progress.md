# TASK-098: Обогащение админ-статистики (дайджест + пост-итоговая сводка)

## Контекст

Расширение базовой админ-статистики из [TASK-097](TASK-097-admin-stats-bot-digest.md). Базовая версия шлёт минимум (всего/новые/прогнозы за сутки; по событию — всего/угадали/распределение + CSV). Владелец хочет более информативные сводки. **Стартовать только после закрытия TASK-097** (использует те же джобы `send_daily_admin_digest` / `dispatch_event_result_notifications`, `StatsService`, тексты).

Никаких новых таблиц/миграций — все данные уже есть (`predictions.created_at`, `events.result_outcome_id`/`archived_at`, `users.created_at`/`last_seen_at`). `last_seen_at` обновляется на каждом апдейте через `UserMiddleware` → DAU считается без новой инфраструктуры.

## Цель

Дополнить дневной дайджест и пост-итоговую сводку метриками, дающими тренд и вовлечённость.

## Definition of Done

> 🚨 **Перед `chore(handoff): archive` — ОБЯЗАТЕЛЬНО `handoff/outbox/TASK-098-report.md`.**

### Дневной дайджест — добавить
- [ ] **Дельта к прошлым суткам** для «новых пользователей» и «прогнозов»: сравнить текущее окно `[now-24h, now)` с предыдущим `[now-48h, now-24h)`, показать `▲ +N` / `▼ −N` / `→ 0`.
- [ ] **DAU за 24ч** — уникальные активные: `count(User where last_seen_at >= now-24h)`. Новый `UserRepository.count_active_since(since)` (или параметризовать существующий `count_active_30d`).
- [ ] **Активных событий сейчас**: `is_published = true AND is_archived = false AND predictions_close_at > now()`.
- [ ] **Топ-3 событий по активности за сутки**: events по числу `predictions.created_at` в окне (title + N). Если за сутки прогнозов не было — строку «нет активности».
- [ ] **Точность закрытых за сутки событий**: события с `archived_at`/`result_outcome_id` в окне 24ч → суммарно `correct/total` прогнозов по ним + %. Если за сутки ничего не закрывали — «нет закрытых событий».
- [ ] **Конверсия новых**: из новых за 24ч сколько сделали ≥1 прогноз (за всё время), `% = converted/new_24h` (гард на `new_24h == 0`).

### Пост-итоговая сводка события — добавить
- [ ] **Самый популярный исход vs верный**: пометить, совпал ли исход с макс. числом прогнозов с верным («большинство угадало / большинство ошиблось»).
- [ ] **% участия**: `total_predictions / total_users` на момент рассылки (гард на 0 пользователей).

### Качество

- [ ] integration-тесты на каждую новую агрегацию (особенно дельта-окна: данные в обоих окнах, граничные значения; точность закрытых — событие закрыто в окне vs вне; конверсия — новый с прогнозом и без).
- [ ] unit-тесты форматирования (стрелки дельты, нулевые случаи, «нет активности»/«нет закрытых»).
- [ ] `ruff`/`mypy src/shared`/`pytest` зелёные.
