# Отчёт по TASK-091: Починить раскрытие деталей аудита + определиться с глобальным поиском

## Сводка

Два дефекта из прод-аудита 2026-05-31 исправлены.

**1. Раскрытие деталей в аудите полностью не работало.**
Кнопки-шевроны имели корректную HTMX-разметку:
```html
hx-get="/audit/{{id}}/details" hx-target="#audit-row-{{id}}" hx-swap="outerHTML"
```
Но в `audit/list.html` ни одна строка таблицы не имела соответствующего элемента (`id="audit-row-..."` отсутствовал вообще). HTMX не находил цель — свап никогда не происходил. Шеврон только локально поворачивался через Alpine, детали не подгружались.

Эндпоинт `/audit/{id}/details` и фрагменты (`_details.html`, `_preview.html`) были написаны правильно и возвращали нужную структуру `<div id="audit-row-...">`.

**Решение:** каждая строка в начальном рендере списка теперь обёрнута в `<div id="audit-row-{{ entry.id }}">` (точно как ожидают фрагменты). Минимальное изменение, сохраняет оригинальный дизайн фрагментов из TASK-026.

**2. Глобальный поиск в топбаре был чисто декоративным.**
Поле «Поиск событий, пользователей, категорий…» в `_layout_shell.html` — отключённый `<input>` без формы, без обработчиков, без бэкенда. Ввод + Enter просто очищал поле.

**Решение (продуктовое):** поле удалено. Реализовывать полноценный кросс-сущностный поиск (события + пользователи + категории + результаты) — объём заметно больше M и выходит за рамки этой задачи. Честнее и дешевле убрать декоративный элемент сейчас (синхронизировано с M3 в TASK-088), чем вводить пользователей в заблуждение. При необходимости — отдельный пункт бэклога.
