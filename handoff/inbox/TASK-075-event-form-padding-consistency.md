# TASK-075: разнобой отступов на форме события (карточки без card-body + хардкод px)

## Контекст (от владельца)

«Отступы не соответствуют на мой взгляд». Подтверждено разбором архитектора.

## Причина

`.pv-card` сам по себе **без внутреннего padding** (только `border`/`radius`/`overflow`).
Инсет даёт `.pv-card-body { padding: var(--pv-s-20) }` **либо** инлайн-стиль.

В `events/form.html` это применено непоследовательно:
- Карточки **«Данные события»** и **«Состояние»** — это `.pv-card` + `.pv-section-title` **без**
  `.pv-card-body` и без своего padding → контент прижат вплотную к рамке (под заголовком только
  `margin-bottom: 12px`).
- Карточки **результата** и **ошибки** используют `.pv-card-body` с инлайном `padding: 20px` /
  `padding: 12px 20px` → нормальный инсет.

Итог: на одной странице карточки с разными внутренними отступами (0 vs 20px). Плюс по шаблону
рассыпан хардкод px (`padding: 10px 16px`, `gap: 8px/10px`, `padding: 12px 16px`), хотя есть
шкала токенов `--pv-s-*` (2/4/6/8/10/12/14/16/20/24/…).
