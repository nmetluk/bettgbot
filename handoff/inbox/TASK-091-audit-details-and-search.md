# TASK-091: Починить раскрытие деталей аудита + определиться с глобальным поиском

## Контекст

Живой прогон админки на проде `a.pinbetting.ru` (2026-05-31, cowork-аудит) нашёл два
функциональных дефекта в журнале и оболочке:

**1. Аудит — раскрытие деталей записи не работает.** У кнопок-шевронов в строках аудита есть
HTMX-разметка:

```
hx-get="/audit/16/details"   hx-target="#audit-row-16"   hx-swap="outerHTML"
```

но **элемента с id `#audit-row-16` (и любым `audit-row-*`) в DOM нет** — у строк таблицы
вообще нет id. HTMX не находит таргет, swap проваливается: шеврон визуально поворачивается
(локальный Alpine-стейт), но детали не подгружаются никогда. Сам endpoint `/audit/{id}/details`,
судя по разметке, существует — ломается только привязка таргета.

**2. Глобальный поиск в топбаре декоративный.** Поле «Поиск событий, пользователей,
категорий…» в `_layout_shell.html` не реагирует: ввод + Enter очищает поле и ничего не делает,
нет перехода к результатам. Совпадает с пунктом **M3** [TASK-088](TASK-088-design-conformance-to-mockup.md)
(«поиск активный + ⌘K-хинт — или явно убрать как нереализованный»).

## Цель

Детали записи аудита раскрываются по клику; по глобальному поиску принято и реализовано
решение — либо рабочий поиск, либо явное удаление поля (чтобы не вводить в заблуждение).
