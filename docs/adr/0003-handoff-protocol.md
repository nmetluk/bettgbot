# ADR 0003 — Handoff-протокол между двумя агентами

**Status:** Accepted
**Date:** 2026-05-22
**Deciders:** cowork-agent, owner

## Context

Над проектом работают два агента:

- **cowork-агент** (десктоп-приложение) — проектирует, пишет спецификации, формирует задачи. Не имеет постоянного процесса и сохранения контекста между сессиями; работает короткими сессиями.
- **локальный Claude Code** — на машине владельца, имеет доступ к git/PAT, исполняет задачи, пушит в GitHub.

Нужен канал связи, который:

1. Сохраняется между сессиями cowork-агента (источник истины — файлы).
2. Не зависит от внешних сервисов (всё в git-репо).
3. Однозначно описывает «что делать», «как понять, что готово», «куда писать отчёт».
4. Даёт человеку (владельцу) возможность в любой момент посмотреть статус.

## Decision

Принят протокол на основе файловой папки **`handoff/`** в репозитории:

```
handoff/
├── inbox/        — задачи от cowork → Claude Code
│   ├── TASK-NNN.md
│   ├── TASK-NNN.in-progress.md   (в работе)
│   └── TASK-NNN.blocked.md       (вопрос)
├── outbox/       — отчёты и вопросы от Claude Code → cowork
│   ├── TASK-NNN-report.md
│   └── TASK-NNN-question.md
├── templates/    — шаблоны task.md, report.md
└── archive/      — закрытые задачи (вместе с отчётами)
```

Жизненный цикл задачи и правила атомарных переходов — в [`../../handoff/README.md`](../../handoff/README.md).

Дополнительные правила:

- Один файл задачи существует в одной папке (переходы через `mv`).
- Cowork-агент не редактирует `inbox/*.in-progress.md` и `outbox/` — только кладёт `amendment.md` если нужно изменить задачу.
- Локальный агент не редактирует `docs/`, `state/`, `sessions/` без явного указания в задаче.
- На каждую закрытую задачу — папка `archive/TASK-NNN-slug/` с `task.md`, `report.md`, опциональными `amendments/`.

## Alternatives considered

| Альтернатива | Почему отвергнуто |
|---|---|
| **GitHub Issues** | Требует онлайна и API-доступа из обоих процессов; теряется автономность; неудобно для длинных детальных задач со ссылками внутри репо |
| **Прямой обмен через чат владельца** | Не сохраняется в репо; невозможно восстановить контекст автоматически; роль владельца усложняется (он становится транспортом) |
| **Только `state/BACKLOG.md` как очередь** | Не описан жизненный цикл задачи (в работе/блок/готово); нет места для отчёта; легко получить рассинхрон |
| **`TASKS.md` с разделами Inbox/Done** | Конфликты merge, сложно атомарно «забрать» задачу в работу |

## Consequences

**Положительные:**

- Полная самодостаточность: всё в git, версионируется, доступно офлайн.
- Простой ментальный модель: «положил файл — передал задачу».
- Лёгкая аудируемость: история задач — это `git log handoff/archive/`.

**Отрицательные:**

- Требует дисциплины именования и переходов (`mv`). Митигация — шаблоны и инструкции в [`CLAUDE.md`](../../CLAUDE.md).
- Нет уведомлений: cowork-агент должен в начале сессии явно проверить `outbox/`. Это формализовано как пункт «начало сессии» в [`feedback-workflow-two-agents`].
