# Решения — task-010-review

| # | Решение | Альтернативы | Обоснование | Где зафиксировано |
|---|---|---|---|---|
| 1 | `src/__init__.py` создан — `src` обычный пакет; все импорты `from src.*` | (а) `mypy_path = "src"` + `explicit_package_bases = true`; (б) рефакторинг shared на относительные импорты везде | (а) сложнее (магия mypy), (б) ломает уже существующий стиль. Однострочный `__init__.py` — минимум магии, и mypy получает одно каноническое имя для каждого модуля. Правило выписано в `docs/08-conventions.md` («`src/` — полноценный пакет, не namespace») | [`docs/08-conventions.md`](../../docs/08-conventions.md), [`state/DECISIONS.md`](../../state/DECISIONS.md) |
| 2 | Throttling `UserMiddleware.touch_last_seen` отложен как тех-долг (не фиксим) | Сразу реализовать Redis-кеш «не чаще раза в N минут» | На MVP нагрузки нет, преждевременная оптимизация. Зафиксировано в новом разделе «Технический долг» в `BACKLOG.md`, триггер — реальные метрики. Когда увидим — реализуем (Redis SETEX user_id → ts + проверка) | [`state/BACKLOG.md`](../../state/BACKLOG.md), [`state/DECISIONS.md`](../../state/DECISIONS.md) |
| 3 | Фабрики и сборочные функции читают конфиг через `get_settings()` (не module-level) | Использовать module-level `settings` везде | Module-level зафиксирован на момент импорта; `cache_clear()` в тестах его не подхватит. Фабрики — `build_dispatcher`, `get_registry_client` — должны видеть свежий env для пересборки. Правило выписано в `docs/08-conventions.md` («Фабрики читают свежий конфиг») | [`docs/08-conventions.md`](../../docs/08-conventions.md), [`state/DECISIONS.md`](../../state/DECISIONS.md) |
| 4 | `structlog.contextvars.clear_contextvars()` в finally `LoggingMiddleware` — оставить | Усиление через task-locals; убрать (полагаться на per-task семантику contextvars) | Python `contextvars` уже per-task в asyncio. Clear — defensive guarantee против shared-loops; стоит ~0 и страхует от рефакторингов | [`src/bot/middlewares/logging.py`](../../src/bot/middlewares/logging.py) |
