# 00 — Обзор проекта

## Что это

«Betting Bot» — Telegram-бот, через который пользователи делают прогнозы на исходы спортивных и иных событий, плюс минимальная веб-админка для управления событиями и просмотра пользовательских прогнозов. Финансовой части нет — это не букмекерский продукт, а инструмент для сбора и учёта прогнозов с автоматической статистикой «сбылся / не сбылся».

## Зачем

Дать закрытому кругу пользователей (проверенных через внешний реестр) удобный способ:

- регулярно делать прогнозы на события из заранее заведённых категорий;
- видеть свою историю и статистику;
- получать напоминания, чтобы не пропускать дедлайны.

Дать администратору:

- быстро заводить события, их возможные исходы и фиксировать итоги;
- видеть пользовательскую активность.

## Scope: что входит

- Регистрация пользователя через передачу контакта Telegram + проверка через внешний API.
- Просмотр активных событий, организованных по категориям-папкам.
- Создание прогноза на событие (один пользователь — один прогноз на событие).
- Просмотр своих прогнозов (активных и архивных).
- Настройка напоминаний (включить/выключить, задать офсеты).
- Команда `/help` со справкой.
- Веб-админка: CRUD категорий, событий, исходов; фиксация итога события; просмотр пользователей и их прогнозов; аудит-лог.
- Архивация прошедших событий (мягкая — скрытие из активных списков, не удаление).
- Контейнеризация и развёртывание через Docker Compose.

## Scope: что НЕ входит

- Любая работа с деньгами: коэффициенты, ставки, выплаты, кошельки, платёжные интеграции. Это не букмекерский продукт.
- Самостоятельная регистрация пользователя (заведение в реестре — задача внешней системы).
- Сложная аналитика / BI: дашборды, графики, выгрузки. На этапе MVP — только базовые списки в админке.
- Многоязычность UI (архитектурно заложено через словарь текстов, но переводы на старте — только русский).
- Мобильное приложение, веб-кабинет пользователя — взаимодействие пользователя идёт только через Telegram-бота.

## Кто пользуется

| Роль | Канал | Что делает |
|---|---|---|
| **Пользователь** | Telegram-бот | Регистрируется, смотрит события, делает прогнозы, смотрит свои результаты, настраивает напоминания |
| **Администратор** | Веб-админка | Заводит категории, события, исходы; фиксирует результаты; смотрит пользователей и прогнозы; читает аудит |
| **Внешняя система-реестр** | HTTP API | Отдаёт ответ «есть ли такой телефон в разрешённых» — на этапе разработки заменена mock-адаптером |

## Ключевые сущности (без деталей; детали — в [03-data-model](03-data-model.md))

- **Категория** (`Category`) — папка событий.
- **Событие** (`Event`) — объект для прогнозирования; в категории; с дедлайном; с возможными исходами; с итогом и флагом архива.
- **Исход** (`Outcome`) — один из вариантов результата события.
- **Прогноз** (`Prediction`) — связка пользователь × событие × выбранный исход; после фиксации итога получает флаг `is_correct`.
- **Пользователь** (`User`) — проверенный по телефону через внешний реестр.
- **Настройки напоминаний** (`ReminderSetting`) — per-user конфигурация.
- **Администратор** (`AdminUser`) — учётка для входа в админку.
- **Аудит-лог** (`AuditLog`) — журнал действий в админке.

Единый словарь терминов — [`state/GLOSSARY.md`](../state/GLOSSARY.md). Использовать только его формулировки.
